# 一页路线图

适用对象：重庆邮电大学 OpenHarmony Hi3861 课程设计学生。

## 目标

完成启航 Hi3861 舒适风扇项目：

```text
温湿度采集
OLED 显示
Wi-Fi 连接
MQTT 双向通信
Web 控制台远程控制
默认 Motor ON 低功率常转与敏感脉冲反馈
噪音较大时可关闭常转
HiBurn 烧录
Xshell 串口验收
```

## 路线

1. 准备资源  
   阅读 [资源获取清单](../../resources/README.md)，确认 Ubuntu、VMware、HiBurn、Mosquitto、Node.js、启航资料包等资源。

2. 配置 Ubuntu 虚拟机  
   用户名建议 `ohos`，工作目录使用 `/home/ohos/workspace`。

3. 拉取并配置 OpenHarmony 3.2 Release  
   核心源码目录：

```text
/home/ohos/workspace/openharmony-3.2-release
```

4. 合并 qihang 适配层并修复旧配置  
   重点是补充 BearPi HAL，删除旧的 `updater/sys_installer_lite` 配置。

5. 首次编译 HelloWorld / qihang  

```bash
cd /home/ohos/workspace/openharmony-3.2-release
hb set -root .
hb set
hb build -f
```

成功标志：

```text
qihang build success
Build result: 0
```

6. 烧录首次固件  
   使用 `Hi3861_wifiiot_app_allinone.bin`，按设备管理器 COM 口、HiBurn、Reset、Disconnect 的流程烧录。

7. Xshell 查看日志  
   `Serial`，波特率 `115200`，按 Reset 查看启动日志。

8. 合入最终项目  
   使用：

```text
comfort-fan-final-package/openharmony_patch/
```

9. 配置 Wi-Fi 和 MQTT  
   把 `network_config.example.h` 复制为 `network_config.h`，填入 Wi-Fi、密码和电脑 MQTT Broker IP。

10. 编译最终固件并烧录  
    最终固件也已放在：

```text
firmware/final/qihang_comfort_fan_mqtt_v6_fast_report_allinone.bin
```

11. 启动 MQTT 和 Web 控制台  

```powershell
Set-Location "C:\Program Files\mosquitto"
.\mosquitto.exe -v
```

```powershell
Set-Location "D:\01_Work\02_class\ohos\comfort-fan-final-package\dashboard"
npm install
npm start
```

浏览器打开：

```text
http://127.0.0.1:8787
```

12. 最终验收  
    确认串口日志、Wi-Fi、MQTT、Web 控制台、OLED、电机控制都正常。

## 最容易出错的点

```text
不要在 /mnt/hgfs/OpenHarmony 中编译源码
不要使用 sudo hb build
不要提交真实 network_config.h
HiBurn 烧录完成后要点 Disconnect
Xshell 串口号要和设备管理器中新出现的 COM 口一致
MQTT_HOST 要填电脑在同一网络下的 IP
```
