# 十五、补充 BearPi HAL 依赖


当前 OpenHarmony 3.2 Release manifest 默认没有 `vendor/bearpi`，因此通过 local manifest 固定补入。

创建配置：

```bash
cd "$HOME/workspace/openharmony-3.2-release"
mkdir -p .repo/local_manifests

cat > .repo/local_manifests/qihang-dependencies.xml <<'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<manifest>
  <remote
      name="qihang-gitee"
      fetch="https://gitee.com/openharmony/" />

  <project
      name="vendor_bearpi"
      path="vendor/bearpi"
      remote="qihang-gitee"
      revision="aa0a2f91e2ec27ea641db7c4748094b817d18599" />
</manifest>
EOF
```

同步：

```bash
repo sync vendor/bearpi -j1 --fail-fast
```

验证：

```bash
git -C vendor/bearpi rev-parse HEAD

test -d vendor/bearpi/bearpi_hm_nano/hals \
  && echo "OK BearPi HAL"
```

预期提交：

```text
aa0a2f91e2ec27ea641db7c4748094b817d18599
```

不要把 `product_adapter_dir` 改为旧版：


# 十七、修复 `sys_installer_lite` 兼容问题

第一次构建可能出现：

```text
Component "sys_installer_lite" not found
```

原因：当前 3.2 Release manifest 中没有：

```text
base/update/sys_installer_lite
```

但启航旧配置仍引用：

```text
updater/sys_installer_lite
```

先备份：

```bash
cd "$HOME/workspace/openharmony-3.2-release"

cp -a \
  vendor/issedu/qihang/config.json \
  vendor/issedu/qihang/config.json.before-remove-sys-installer-lite
```

精确删除：

```bash
python3 - <<'PY2'
import json
from pathlib import Path

path = Path("vendor/issedu/qihang/config.json")
data = json.loads(path.read_text(encoding="utf-8"))

removed = False
new_subsystems = []

for subsystem in data.get("subsystems", []):
    if subsystem.get("subsystem") != "updater":
        new_subsystems.append(subsystem)
        continue

    original = subsystem.get("components", [])
    remaining = [
        item for item in original
        if item.get("component") != "sys_installer_lite"
    ]

    if len(remaining) != len(original):
        removed = True
        print("Removed component: updater/sys_installer_lite")

    if remaining:
        subsystem["components"] = remaining
        new_subsystems.append(subsystem)
    else:
        print("Removed empty subsystem: updater")

if not removed:
    raise SystemExit("ERROR: sys_installer_lite was not found")

data["subsystems"] = new_subsystems
path.write_text(
    json.dumps(data, ensure_ascii=False, indent=2) + "\n",
    encoding="utf-8",
)
PY2
```

验证：

```bash
python3 -m json.tool \
  vendor/issedu/qihang/config.json \
  >/dev/null \
  && echo "JSON OK"
```

```bash
grep -nE \
  'sys_installer_lite|"subsystem"[[:space:]]*:[[:space:]]*"updater"' \
  vendor/issedu/qihang/config.json
```

正常应无输出。

# 二十六、常见问题

## 1. `Component "sys_installer_lite" not found`

原因：启航旧配置引用了当前 3.2 Release 不存在的组件。

处理：备份并删除 `updater/sys_installer_lite`。

## 2. `riscv32-unknown-elf-gcc: not found`

检查：

```bash
command -v riscv32-unknown-elf-gcc
```

处理：

```bash
export PATH="$HOME/tools/gcc_riscv32/bin:$PATH"
source ~/.bashrc
hash -r
```

## 3. `ccache disabled`

只影响重复编译速度，不影响固件正确性。

## 4. 不要误粘贴终端内容

不要复制：

```text
ohos@ohos-vm:~$
>
OK
MISSING
```

只复制命令本身。

## 5. 不要使用 sudo 构建

不要执行：

```bash
sudo hb build
sudo bash build/prebuilts_download.sh
```

## 6. 不要在共享目录编译

```text
/mnt/hgfs/OpenHarmony
```

只用于传输固件和日志。
