## 二、选题背景与意义

随着物联网和智能家居技术的发展，家庭环境安全与舒适度监测逐渐成为智能设备应用的重要场景。传统环境检测设备通常功能单一，数据只能在本地查看，缺少远程监控和异常提醒能力。本课题基于 OpenHarmony 轻量系统和 Hi3861 开发板，设计一个能够采集温湿度、烟雾或可燃气体数据，并支持 OLED 本地显示、异常报警和 MQTT 云端上报的智能家居环境监测系统。

本课题能够综合训练嵌入式设备开发、传感器数据采集、外设通信、物联网协议和系统集成能力，符合硬件综合设计开发课程对软硬件结合、工程实践和团队协作的要求。

## 三、研究现状

当前智能家居环境监测产品主要包括温湿度计、烟雾报警器、燃气报警器和多功能环境监测终端。商业产品通常具备无线联网、手机 App 查看和异常告警功能。教学场景中的 OpenHarmony 设备开发可以利用 Hi3861 的 Wi-Fi 能力和轻量系统接口，实现类似的物联网终端功能。

本项目选择温湿度、烟雾/可燃气体和 OLED 显示作为核心模块，能够覆盖 I2C、ADC、SPI/GPIO、Wi-Fi 和 MQTT 等常见嵌入式开发知识点，具有较好的教学价值和可实现性。

## 四、可行性分析

### 4.1 技术可行性

项目使用 Hi3861 开发板作为主控，启航 KP-IOT 套件中已经提供 OLED、SHT30、烟雾报警、可燃气体等模块源码和参考资料。系统可以通过 OpenHarmony 设备端 API 完成 GPIO、I2C、ADC、Wi-Fi 和 MQTT 相关功能。已有模块可以先单独调试，再逐步集成，技术路径清晰。

### 4.2 经济可行性

项目主要使用课程提供的开发板和传感器模块，软件工具包括 DevEco Device Tool、串口调试工具和云平台控制台，整体成本较低，适合课程设计周期内完成。

### 4.3 操作可行性

小组可以按照“环境搭建、单模块调试、系统集成、云平台接入、测试答辩”的流程推进。即使云平台接入遇到问题，也可以保证本地显示和报警作为可演示的核心功能。

## 五、需求分析

### 5.1 功能需求

| 编号 | 功能模块 | 功能描述 |
|---|---|---|
| F1 | 温湿度采集 | 通过 SHT30 周期采集室内温度和湿度 |
| F2 | 气体/烟雾检测 | 通过 ADC 读取可燃气体或烟雾传感器数据 |
| F3 | 本地显示 | 通过 OLED 显示温度、湿度、气体/烟雾值和系统状态 |
| F4 | 阈值报警 | 当温度、湿度或气体/烟雾值超过阈值时显示报警状态并触发本地报警 |
| F5 | 云端上报 | 通过 MQTT 将环境数据和报警状态上传至云平台 |
| F6 | 远程查看 | 在云平台查看设备在线状态、实时数据和历史数据 |

### 5.2 性能需求

| 项目 | 指标 |
|---|---|
| 数据采集周期 | 1 秒左右采集一次 |
| OLED 刷新周期 | 1 秒左右刷新一次 |
| MQTT 上报周期 | 5 秒左右上报一次 |
| 报警响应时间 | 不超过 2 秒 |
| 连续运行时间 | 支持 30 分钟以上稳定运行 |

### 5.3 约束条件

- 开发周期有限，应优先保证本地采集、显示和报警功能。
- Hi3861 资源有限，程序结构应保持简洁。
- MQTT 功能依赖 Wi-Fi 和云平台配置，需预留调试时间。
- 现场演示需准备离线方案，避免网络不稳定影响答辩。
