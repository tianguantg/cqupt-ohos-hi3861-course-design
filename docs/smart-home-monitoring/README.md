## 推荐开发路线

1. 跑通 `code-master.zip` 的 HelloWorld，验证编译、烧录和串口输出。
2. 跑通 `022_OH_KP_OLED_SAMPLE`，先完成本地显示。
3. 调试 SHT30 温湿度采集，并在串口输出数据。
4. 调试烟雾或可燃气体 ADC 采集。
5. 整合 OLED、温湿度、气体检测和阈值报警。
6. 接入 MQTT 云平台，上报传感器数据和报警状态。
7. 整理测试截图、串口日志、云平台截图和答辩材料。

## 最小可演示版本

最小版本建议包含以下功能：

- OLED 显示温度、湿度、气体/烟雾数值和系统状态。
- SHT30 周期采集温湿度数据。
- 可燃气体或烟雾传感器读取环境异常值。
- 超过阈值时显示 `Warning` 状态并触发本地报警。
- 串口输出关键运行日志，便于答辩现场说明。

MQTT 云平台属于完整版本功能，建议在本地功能稳定后再接入。
