# 项目说明

本项目是基于 OpenHarmony Hi3861 的智能舒适风扇系统。设备端通过 SHT30 采集环境温湿度，在 OLED 上显示状态，并通过 MQTT 向电脑端 Broker 上报状态、接收远程控制命令。

## 控制逻辑

- OFF：目标风速 0%。
- QUIET：目标风速 25%，ECO 后为 15%。
- AUTO：温度决定基础风速，湿度提供补偿。

AUTO 温度曲线：

| 温度 | 基础目标 |
| --- | ---: |
| < 26°C | 0% |
| 26-27.4°C | 30% |
| 27.5-28.9°C | 45% |
| 29-30.9°C | 60% |
| >= 31°C | 75% |

湿度补偿：

| 湿度 | 补偿 |
| --- | ---: |
| > 65% | +5% |
| > 75% | +10% |

ECO 和偏好：

- ECO_ON：偏置 -10%。
- ECO_OFF：偏置 0%。
- MORE / LESS：调整偏好偏置。

虚拟风速：

- 每次最多变化 5%。
- MQTT 状态约 1 秒上报一次。
- 等级：0=OFF，1-35=LOW，36-60=MEDIUM，61+=HIGH。

电机策略：

- 默认策略：电机常转开启，虚拟风速 > 0 时以最低占空比持续转动。
- 敏感脉冲反馈：虚拟等级变化或虚拟风速每变化 5% 时短暂脉冲。
- MOTOR_ON：开启低占空比常转。
- MOTOR_OFF：关闭常转，但保留敏感脉冲反馈。
- 如果实机演示时电机常转噪音太大，可以发送 MOTOR_OFF，或在自定义前端中提供“关闭常转”按钮，保留状态反馈但降低持续噪音。
