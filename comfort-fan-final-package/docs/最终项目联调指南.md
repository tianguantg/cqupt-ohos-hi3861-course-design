# 最终项目联调步骤

这篇是基础环境跑通之后继续做最终项目用的。我把“合入代码、重新编译、烧录、MQTT 验证、前端控制”串成一条线，尽量避免做到一半不知道下一步该看哪里。

开始前先确认已经完成 [通用配置步骤](../../docs/course-design/01_通用配置步骤.md)，至少做到 `qihang build success`、HiBurn 烧录成功、Xshell 能看到串口日志。

## 1. 我建议的联调顺序

```text
1. 复制最终项目代码到 OpenHarmony
2. 配置 network_config.h
3. 编译并烧录固件
4. 启动 Mosquitto MQTT Broker
5. 用 mosquitto_sub / mosquitto_pub 验证 MQTT
6. 基于 simple-dashboard 编写或改造前端
7. 可选：阅读完整 dashboard 参考实现
8. 完成最终验收
```

| 编译、复制固件、HiBurn、Xshell | [编译烧录指南](./编译烧录指南.md) |

## 3. 最小可用命令

编译：

```bash
export PATH="$HOME/.local/bin:$HOME/tools/gcc_riscv32/bin:$PATH"
cd /home/ohos/workspace/openharmony-3.2-release
hb build -f
```

启动 Mosquitto：

```powershell
Set-Location "C:\Program Files\mosquitto"
.\mosquitto.exe -v
```

订阅状态：

```powershell
.\mosquitto_sub.exe -h 127.0.0.1 -p 1883 -t "comfortfan/qihang01/#" -v
```

发送命令：

```powershell
.\mosquitto_pub.exe -h 127.0.0.1 -p 1883 -t "comfortfan/qihang01/cmd" -m "AUTO"
.\mosquitto_pub.exe -h 127.0.0.1 -p 1883 -t "comfortfan/qihang01/cmd" -m "MOTOR_OFF"
```

前端开发起点：

```text
templates/simple-dashboard/
```

完整 dashboard 仅作为参考实现，需要时再阅读：

```text
comfort-fan-final-package/dashboard/
```

## 4. 最终验收标准

```text
1. 固件编译成功，出现 qihang build success
2. HiBurn 烧录成功，烧录后已 Disconnect
3. Xshell 115200 串口能看到设备启动日志
4. 设备能连接 Wi-Fi，串口出现 Wi-Fi link connected / DHCP ready
5. 设备能连接 MQTT，串口出现 MQTT connected
6. mosquitto_sub 能收到 state 和 event
7. mosquitto_pub 发送 AUTO / QUIET / OFF 后设备状态变化
8. 自己编写或改造的前端能生成或发送控制命令
9. 前端能显示温湿度、虚拟风速、模式、ECO、Motor Hold 等核心状态
10. 默认 Motor ON 低功率常转；噪音较大时可用 MOTOR_OFF 降噪
11. OLED 能显示当前模式、风速、Wi-Fi/MQTT 状态
```
