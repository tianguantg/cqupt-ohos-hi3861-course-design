# 最终项目联调指南

本文把最终项目从代码、固件、MQTT 到 Web 控制台串成一条完整路线。建议按顺序执行，不要跳步。

前置条件：已经完成 [通用配置步骤](../../docs/course-design/01_通用配置步骤.md)，至少做到 `qihang build success`、HiBurn 烧录成功、Xshell 能看到串口日志。第一次配置环境的同学不要从本文开始。

## 1. 联调顺序

```text
1. 复制最终项目代码到 OpenHarmony
2. 配置 network_config.h
3. 编译并烧录固件
4. 启动 Mosquitto MQTT Broker
5. 用 mosquitto_sub / mosquitto_pub 验证 MQTT
6. 启动 Web 控制台
7. 完成最终验收
```

对应文档：

| 步骤 | 文档 |
|------|------|
| 编译、复制固件、HiBurn、Xshell | [编译烧录指南](./编译烧录指南.md) |
| Broker、主题、命令、串口日志 | [MQTT 测试指南](./MQTT测试指南.md) |
| Node.js、Web 页面、按钮、环境变量 | [Web 控制台使用指南](./Web控制台使用指南.md) |
| 实机检查记录 | [实机验收记录](./实机验收记录.md) |

## 3. 最小可用命令

编译：

```bash
export PATH="$HOME/.local/bin:$HOME/tools/gcc_riscv32/bin:$PATH"
cd /home/ohos/workspace/openharmony-3.2-release
hb build -f
```

启动 Mosquitto：

```powershell
Set-Location "C:\Program Files\mosquitto"
.\mosquitto.exe -v
```

订阅状态：

```powershell
.\mosquitto_sub.exe -h 127.0.0.1 -p 1883 -t "comfortfan/qihang01/#" -v
```

发送命令：

```powershell
.\mosquitto_pub.exe -h 127.0.0.1 -p 1883 -t "comfortfan/qihang01/cmd" -m "AUTO"
.\mosquitto_pub.exe -h 127.0.0.1 -p 1883 -t "comfortfan/qihang01/cmd" -m "MOTOR_OFF"
```

启动 Web：

```powershell
Set-Location "C:\OpenHarmony\cqupt-ohos-hi3861-course-design\comfort-fan-final-package\dashboard"
npm install
npm start
```

打开：

```text
http://127.0.0.1:8787
```

## 4. 最终验收标准

```text
1. 固件编译成功，出现 qihang build success
2. HiBurn 烧录成功，烧录后已 Disconnect
3. Xshell 115200 串口能看到设备启动日志
4. 设备能连接 Wi-Fi，串口出现 Wi-Fi link connected / DHCP ready
5. 设备能连接 MQTT，串口出现 MQTT connected
6. mosquitto_sub 能收到 state 和 event
7. mosquitto_pub 发送 AUTO / QUIET / OFF 后设备状态变化
8. Web 控制台能打开并显示 Broker online
9. Web 控制台能显示温湿度、虚拟风速、模式、ECO、Motor Hold
10. Web 控制台按钮能控制设备
11. 默认 Motor ON 低功率常转；噪音较大时可用 MOTOR_OFF 或“关闭常转”按钮降噪
12. OLED 能显示当前模式、风速、Wi-Fi/MQTT 状态
```
