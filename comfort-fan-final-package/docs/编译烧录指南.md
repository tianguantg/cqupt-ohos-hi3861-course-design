# 编译与烧录指南

本文负责把最终项目代码放入 OpenHarmony，重新编译固件，并用 HiBurn 烧录到启航 Hi3861 开发板。MQTT 和 Web 控制台联调见后续文档。

## 1. 文件对应关系

最终项目代码在：

```text
comfort-fan-final-package/openharmony_patch/
├── fan_manager.c
├── BUILD.gn
└── network_config.example.h
```

复制到 OpenHarmony 应用目录：

```text
/home/ohos/workspace/openharmony-3.2-release/device/board/issedu/qihang/app/fan_manager/
```

三个文件作用：

| 文件 | 作用 |
|------|------|
| `fan_manager.c` | 设备端主程序，包含 OLED、SHT30、风扇控制、Wi-Fi、MQTT 命令处理 |
| `BUILD.gn` | OpenHarmony 构建配置 |
| `network_config.example.h` | 网络配置模板，复制为 `network_config.h` 后填写真实 Wi-Fi 和 MQTT 地址 |

## 2. 复制项目代码

在 Ubuntu 中执行：

```bash
export ROOT="/home/ohos/workspace/openharmony-3.2-release"
export PKG="$HOME/workspace/cqupt-ohos-hi3861-course-design/comfort-fan-final-package"
export APP_DIR="$ROOT/device/board/issedu/qihang/app/fan_manager"

mkdir -p "$APP_DIR"
cp "$PKG/openharmony_patch/fan_manager.c" "$APP_DIR/"
cp "$PKG/openharmony_patch/BUILD.gn" "$APP_DIR/"
cp "$PKG/openharmony_patch/network_config.example.h" "$APP_DIR/network_config.h"
```

检查：

```bash
ls -la "$APP_DIR"
```

应能看到：

```text
BUILD.gn
fan_manager.c
network_config.h
```

## 3. 配置 Wi-Fi 和 MQTT

编辑：

```bash
nano "$APP_DIR/network_config.h"
```

模板内容类似：

```c
#define CF_WIFI_SSID       "YOUR_WIFI_SSID"
#define CF_WIFI_PASSWORD   "YOUR_WIFI_PASSWORD"
#define CF_MQTT_URI        "tcp://192.168.137.1:1883"
#define CF_MQTT_USERNAME   ""
#define CF_MQTT_PASSWORD   ""
#define CF_DEVICE_ID       "qihang01"
```

按实际环境修改：

```c
#define CF_WIFI_SSID       "你的WiFi名称"
#define CF_WIFI_PASSWORD   "你的WiFi密码"
#define CF_MQTT_URI        "tcp://电脑IP地址:1883"
#define CF_MQTT_USERNAME   ""
#define CF_MQTT_PASSWORD   ""
#define CF_DEVICE_ID       "qihang01"
```

常见写法：

| 场景 | `CF_MQTT_URI` 示例 |
|------|--------------------|
| Windows 开手机热点/移动热点 | `tcp://192.168.137.1:1883` |
| 电脑和开发板在同一 Wi-Fi | `tcp://电脑局域网IP:1883` |
| MQTT Broker 在另一台电脑 | `tcp://Broker电脑IP:1883` |

Windows 查看电脑 IP：

```powershell
ipconfig
```

注意：

```text
network_config.h 包含真实 Wi-Fi 密码，不要提交到公开仓库。
如果换 Wi-Fi、换电脑 IP 或换 MQTT Broker，需要重新修改 network_config.h 并重新编译烧录。
```

## 4. 确认代码入口

设备端主程序入口在：

```text
fan_manager.c
```

关键位置：

| 代码位置 | 说明 |
|----------|------|
| `#include "network_config.h"` | 引入 Wi-Fi 和 MQTT 配置 |
| `BuildMqttTopics()` | 根据 `CF_DEVICE_ID` 生成 MQTT 主题 |
| `ConnectWifi()` | 连接 Wi-Fi |
| `ConnectMqtt()` | 连接 MQTT Broker 并订阅命令主题 |
| `MqttMessageArrived()` | 接收 MQTT 命令 |
| `HandleRemoteCommand()` | 处理 `OFF`、`AUTO`、`MOTOR_OFF` 等命令 |
| `PublishFanState()` | 发布温湿度、风速、模式等 JSON 状态 |

默认主题由 `CF_DEVICE_ID` 决定：

```text
comfortfan/qihang01/state
comfortfan/qihang01/cmd
comfortfan/qihang01/event
```

如果修改：

```c
#define CF_DEVICE_ID "team01"
```

主题会变为：

```text
comfortfan/team01/state
comfortfan/team01/cmd
comfortfan/team01/event
```

## 7. HiBurn 烧录

打开设备管理器：

```text
端口（COM 和 LPT）
```

使用 Type-C 数据线连接电脑和开发板，记录新出现的 COM 口，例如 `COM3`。

HiBurn 操作：

```text
打开 resources/tools/HiBurn.exe
选择设备管理器中新出现的 COM 口
点击 Select file，选择 Hi3861_wifiiot_app_allinone.bin
勾选 Auto burn
点击 Connect
按下开发板 Reset
等待烧录完成
点击 Disconnect
```

烧录完成后必须点击 `Disconnect`，否则 Xshell 可能无法打开同一个串口。

## 8. Xshell 串口验证

Xshell 新建会话：

```text

端口号：设备管理器中新出现的 COM 口
波特率：115200
数据位：8
停止位：1
校验：None
流控：None
```

连接后按开发板 `Reset`。

正常日志应包含：

```text
[COMFORT_FAN] initializing simulated fan v6 + MQTT
[COMFORT_FAN] boot mode=OFF; motor hold defaults to ON
[NET] network task started
[NET] state topic=comfortfan/qihang01/state
[NET] command topic=comfortfan/qihang01/cmd
[NET] Wi-Fi link connected
[NET] DHCP ready
[NET] MQTT connected; subscribed=comfortfan/qihang01/cmd
```

如果只看到应用启动，没有看到 `MQTT connected`，继续阅读 [MQTT 测试指南](./MQTT测试指南.md) 排查 Broker、IP、防火墙和主题。
